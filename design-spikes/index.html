Design Spikes Hub

This is the entry point for the spike-first workflow: shape the full presentation and interaction behavior here, lock it with a contract, then apply it cleanly in React.

Design first Contract second React translation third State driven + event mutated

Flow

1. Spike

Build the panel or visor behavior in plain HTML/CSS/JS until the interaction feels right.

2. Enumerate States

Cover default, loading, empty, error, auth-required, and edge behavior before implementation.

3. Contract

Define expected behavior in design-spikes/contracts/ and keep it explicit.

4. Apply

Translate into React with no redesign drift, then validate against contract tests.
